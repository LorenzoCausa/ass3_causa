Specifies the detection mode. We have preset three types of detection modes. These are ways to configure the internal parameters for the most typical situations. The modes are\+:
\begin{DoxyItemize}
\item D\+M\+\_\+\+N\+O\+R\+M\+AL\+: In this mode, the full resolution image is employed for detection and slow threshold method. Use this method when you process individual images that are not part of a video sequence and you are not interested in speed.
\item D\+M\+\_\+\+F\+A\+ST\+: In this mode, there are two main improvements. First, image is threshold using a faster method using a global threshold. Also, the full resolution image is employed for detection, but, you could speed up detection even more by indicating a minimum size of the markers you will accept. This is set by the variable min\+Marker\+Size which should be in range \mbox{[}0,1\mbox{]}. When it is 0, means that you do not set a limit in the size of the accepted markers. However, if you set 0.\+1, it means that markers smaller than 10\% of the total image area, will not be detected. Then, the detection can be accelerated up to orders of magnitude compared to the normal mode.
\item D\+M\+\_\+\+V\+I\+D\+E\+O\+\_\+\+F\+A\+ST\+: This is similar to D\+M\+\_\+\+F\+A\+ST, but specially adapted to video processing. In that case, we assume that the observed markers when you call to \mbox{\hyperlink{classaruco_1_1_marker_detector_adfb8f5e750be70433fff38e7270aaf82}{detect()}} have a size similar to the ones observed in the previous frame. Then, the processing can be speeded up by employing smaller versions of the image automatically calculated.
